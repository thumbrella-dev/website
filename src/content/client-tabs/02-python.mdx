The Python client provides easy, high level functionality for caching, error
control, and other conveniences.

Use `uv install thumbrella/client` to get started (or the equivalent
for your package manager of choice).

The Python interface provides both a syncronous API (which relies on `requests`)
and an async streaming call (which relies on `aiohttp`).

```python
import thumbrella

tbr = thumbrella.Client()
result = tbr.thumb("https://demo.thumbrella.dev/media/harbor-trucks.mp4")

print(f"Media type: {result.kind}")
Path("thumbnail.jpeg").write_bytes(result.thumbnail)

from PIL import Image
img = Image.open(result.thumbnail.io)
```

The connection string is usually an http server url, or an authentication
token to use with the standard hosted server. It can also be set to the
`$TBR_CONNECT` environment variable to get used automatically.

The `Result.thumbnail` value provides efficient buffer and io objects allow
decode data into image libraries like `Pillow` or `Qt` without needing to 
import or depend on those libraries itself.

The ``async`` method ``stream()`` returns a series of incremental updates as
thumbnails and placeholders are available.
